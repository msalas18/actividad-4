Actividad 4

Comando Descripción Parámetros Ejemplo
git config Establecer una configuración inicial para cuando se revise el historial de versiones --global: se refiere a que será una configuración global
-e: Fichero con la información de configuración de Git		 git config −−global user.name “[nombre-apellido]”
git init Crear un repositorio de Git vacío o reinicializar uno ya existente --q: Crear un repositorio de Git vacío o reinicializar uno ya existente
git init clonar un repositorio en un directorio nuevo --local
git add Añadir contenido de un fichero al índice --dry-run: Realmente no añade los ficheros. Sólo muestra si existen o se ignorarán. git add --dry-run .
git status Revisa el estado de los archivos en el repositorio. --short: produce una salida más concisa y fácil de leer que muestra solo el estado de los archivos en un formato compacto. git status -s
git commit Guarda los cambios en el repositorio. -m: agrega un comentario al cambio realizado. git commit -m "Primer commit"
git restore Guarda los cambios en el repositorio. --staged: agrega un comentario al cambio realizado. git restore --staged
git reset Mueve el puntero HEAD y, opcionalmente, el puntero de una rama, a un commit específico. --soft: Reestablece el HEAD al commit deseado, pero mantiene los cambios realizados en el área de staging (índice) y en el directorio de trabajo sin cambios. git reset --soft
git rm Elimina archivos del repositorio, y del sistema de archivos local al mismo tiempo. --cached: Quita los archivos del área del índice pero los dejará en el sistema de archivos local. git rm --cached nombre_del_archivo
git mv Mover o renombrar archivos y directorios en tu repositorio Git. -- git mv archivo_viejo.txt archivo_nuevo.txt
git branch Crear, listar y administrar ramas en un repositorio Git. -d: eliminar una rama
-m: cambiar el nombre de una rama.		 git branch -m nombre_nuevo nombre_antiguo
git checkout Permite cambiar entre ramas o commits en un repositorio. -b: permite crear una nueva rama y cambiar automáticamente a ella. git checkout -b nombre_de_la_rama
git switch Permite cambiar entre ramas o commits en un repositorio (es una versión actualizada de checkout). -c: permite crear una nueva rama y cambiar automáticamente a ella. git checkout -c nombre_de_la_rama
git merge Fusionar cambios de una rama en otra. --abort: Restaura el estado anterior a la fusión en caso de que haya un conflicto. git merge --abort
git tag Crea etiquetas (tags) que permiten marcar puntos específicos en la historia de un repositorio. -d: elimina el nombre de una etiqueta. git tag v1.0
git worktree Permite crear y gestionar directorios de trabajo adicionales asociados a un mismo repositorio. list: Lista todos los directorios de trabajo asociados al repositorio. git worktree list
git fetch Recuperar cambios remotos (commits, ramas, etiquetas, etc.) de un repositorio remoto al repositorio local sin realizar ninguna fusión automática o modificación en la rama actual. --all: Este parámetro recupera todas las ramas remotas y etiquetas del repositorio remoto. git fetch --all
git pull Actualiza el repositorio local con los cambios del repositorio remoto. --rebase: aplica los cambios locales sobre los cambios del repositorio remoto. git pull --rebase origin rama
git push Envía tus cambios locales desde tu repositorio local a un repositorio remoto --force: forza los cambios cuando los historiales divergen o se desea sobreescribir la rama remota con la versión local. git push --force origin rama
git remote Gestionar y ver información sobre los repositorios remotos asociados con el repositorio local. add: agrega un nuevo repositorio remoto. git remote add
git show Ver información detallada sobre un objeto específico en la historia de un repositorio. --stat: muestra estadísticas de cambios en los archivos afectados por el commit, incluyendo la cantidad de inserciones y eliminaciones. git show --stat
git diff Mostrar los cambios entre commits, commits y el árbol de trabajo, etc. --name-only: Muestra solo los nombres de los archivos que son diferentes, sin mostrar las diferencias reales. git diff --name-only
git log Proporciona información detallada sobre las commits, como los autores, las fechas, los mensajes de confirmación y los identificadores únicos (hash) de los commits. --author: filtra las confirmaciones por autor específico. git log --author="msalas"
git describe Proporcionar una descripción amigable y legible por humanos de un commit específico en el historial de Git en relación con las etiquetas (tags) más cercanas. --abbrev: permite especificar la cantidad de caracteres a utilizar en el hash abreviado de la etiqueta. git describe --abbrev=7 --dirty HEAD
git apply Aplicar cambios contenidos en un parche (patch) al directorio de trabajo o al índice (staging area) de Git. --check: verifica si el parche se puede aplicar con éxito sin realizar ningún cambio en los archivos del proyecto. git apply --check
git cherry-pick Aplicar un commit específico de una rama a otra. Esto significa que se pueden tomar los cambios introducidos por un commit en una rama y aplicarlos a otra rama sin tener que fusionar toda la rama. --edit: permite editar el mensaje de confirmación del commit antes de realizar el cherry-pick. Después de ejecutar el comando, se abrirá el editor de texto para modificar el mensaje. git cherry-pick -e
git revert Deshacer cambios introducidos por un commit o una serie de commits específicos en una rama. --no-commit: realiza el proceso de revertido pero no crea automáticamente un nuevo commit. Esto te permite realizar cambios adicionales o ajustes antes de confirmar el revertido. git revert --no-commit
git bisect Realizar una búsqueda binaria en el historial de commits para encontrar el commit exacto que introdujo un problema o un cambio no deseado en el proyecto. --start: especifica dos commits conocidos: uno en el que el problema no existe y otro en el que el problema está presente. git bisect start
git blame Muestrala autoría de cada línea en un archivo específico dentro de un repositorio. Proporciona información detallada sobre quién realizó la última modificación en cada línea de un archivo, junto con el hash SHA-1 del commit en el que se realizó la modificación y la fecha en que se hizo -L: permite especificar un rango de líneas en el archivo que deseas analizar. git blame -L 10,20
git grep Buscar y filtrar contenido en archivos del repositorio. Permite buscar patrones de texto específicos dentro de los archivos del proyecto. -i: realiza la búsqueda de manera insensible a mayúsculas y minúsculas, lo que significa que coincidirá con el patrón sin importar si las letras son mayúsculas o minúsculas. git grep -i
Conclusión
Git es una herramienta bastante útil en el control de versiones, ya que cuenta con una amplia
variedad de funciones que permiten trabajar de manera colaborativa de forma muy cómoda.
Aunque en la tabla anterior no se muestran TODOS los comandos de Git, se puede observar que
da muchas opciones para el manejo de directorios, cubriendo la mayoría de las necesidades
que se pueden presentar al trabajar colaborativamente.
